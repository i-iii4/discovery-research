# discovery-research

Экосистема мультиагентных пайплайнов для поиска жизнеспособных продуктовых возможностей (B2C и B2B) через LLM-агентов в Claude Code.

## Идея

Ручной поиск идей субъективен и медленный (founder bias). Эти пайплайны автоматизируют генерацию и отсев гипотез через цепочки LLM-агентов, заземляя их либо на живых сигналах спроса из веба, либо на симуляции глубинных JTBD-интервью.

## Два подхода (фазы)

| Фаза | Подход | Источник правды | Что ловит |
|---|---|---|---|
| **01 — market** | web-grounded анализ рынка | живые сигналы спроса (форумы, отзывы, сторы) | что рынок уже сигналит |
| **02 — jtbd** | симуляция глубинных интервью | смоделированные интервью (Mom Test + AJTBD) | глубину и остроту боли |

Подходы комплементарны: первый заземляет на реальности, второй стресс-тестит спрос там, где открытых сигналов нет.

## Матрица пайплайнов

```
                B2C                     B2B
        ┌───────────────────┬───────────────────┐
Фаза 01 │ 01-market-b2c  ✓  │ 01-market-b2b  ○  │
market  │                   │                   │
        ├───────────────────┼───────────────────┤
Фаза 02 │ 02-jtbd-b2c    ✓  │ 02-jtbd-b2b     ○  │
jtbd    │                   │                   │
        └───────────────────┴───────────────────┘
        ✓ работает   ○ планируется (форк B2C)
```

## Структура репозитория

```
discovery-research/
  pipelines/              # методология (промпты, правила, архитектура) — публикуется
    01-market-b2c/
    02-jtbd-b2c/
  profiles/               # профили основателя (saas, mobile-us) — публикуется
  templates/              # эталонные схемы артефактов (job, hypothesis) — публикуется
  artifacts/              # данные прогонов (jobs, гипотезы, интервью, логи) — локально, в .gitignore
```

**Принцип разделения:** `pipelines/` — это методология (read-only машинерия, под git), `artifacts/` — это данные, которые произвели прогоны (вне git, локально).

**Единый обогащаемый артефакт.** Все пайплайны пишут в общую папку `artifacts/` по эталонам из `templates/`. Один артефакт = одна работа или гипотеза, обогащаемая несколькими пайплайнами: market дописывает блок `## Доказательства — market`, jtbd — `## Доказательства — jtbd`, а список `pipelines` в frontmatter копит, кто её трогал. Принадлежность задаётся атрибутами `audience: b2c|b2b`, `pipelines: [market, jtbd]`, `profile`. Пересечение методов — главная ценность: артефакт, подтверждённый обоими, — сильнейший сигнал.

## Профили прогонов

Один пайплайн запускается под разные профили основателя через атрибут `profile`. Текущие:

- `saas` — исходный профиль (solo-разработчик, web/SaaS)
- `mobile-us` — мобильные приложения под рынок США, подписочная монетизация, расширенная рамка (код — commodity, главный фильтр — органическая дистрибуция без UA-бюджета)

## Статус

- **Этап A — реструктуризация** ✓ — единый корень, разделение методология/данные, git
- **Этап B — контент** ○ — адаптация B2C-пайплайнов под `mobile-us`, унификация атрибутов артефактов
- **Этап C — расширение** ○ — B2B-форки пайплайнов
